Death Valley

The Problem

I needed to create two unique pages for my website called Death Valley for the web development class at MSU. I choose to create two pages: the drawing page and the trivia page. The drawing page was a mini MS paint in the browser while the trivia page allowed the user to pick a category and then answer trivia questions which the web page would let them know what they got right or wrong. I also customized the website by adding ‘silly mode’ which I have implemented in this website as well… It’s up to you to find out what it does :)

The 10 Usability Heuristics

Visibility of system status

On the Trivia page while the page is grabbing and generating the questions from the trivia API the website displays “Awaiting choices…” to show the user that it is working.

Match between system and real world

On the drawing page everything's made to match what it would be in the real world, particularly the canvas. The canvas is a white rectangle that looks like a sheet of paper on its side.

User control and freedom

The drawing page has a clear button so the user can always get rid of their drawing if they don’t like it. The nav panel is always accessible from the side so the user is always able to exit or change the page they are on.

Consistency and Standards

All of the choices for both the drawing and trivia page match with other systems. They are a combination of dropdown menus, radio buttons, and one color choice panel. This is consistent with other websites and with MS paint.

Error Prevention

In order to add in error prevention on my page I made it so that all the choices had choices filled in already. This made it so I knew all the features would work even if the user forgot to make a choice. For example here is the draw page with all the default choices.

Recognition rather than recall

The choices on both of my pages have labels for all the options so the user doesn’t have to try and remember what each thing does.

Flexibility and efficiency of use

The trivia page allows users to choose what kinds of questions they want to answer but it also has the categories filled in so the user can just jump straight into the trivia if they want. This lets more advanced users customize their experience while letting novices just jump right in.

Aesthetic and minimalist design

I stuck to a few colors for my website design. Everything that was on my webpages was necessary to my site. There was minimal clutter while maintaining a pleasant appearance.

Recognize and recover from errors

There aren’t any errors that can happen on my webpages but if I could go back in and change the website I would add a pop-up on the trivia page for when users do not answer all the questions on the page. It would look a little bit like the example I provided on the right.

Help and Documentation

On the homepage of the website is a little bit about each of the pages and features that I added. The Most helpful part is the bit about ‘Silly mode’ which tells the user what it does and how the user should engage ‘Silly mode’
